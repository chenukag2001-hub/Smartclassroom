Research Domain

Scope & Background

A deep-dive into the problem space, existing work, gaps identified, and the methodology underpinning this research.

Literature Survey

State of the Art

Modern classroom technologies mainly focus on isolated functionalities such as environmental monitoring or attendance tracking. Cloud-based systems dominate current solutions, introducing privacy risks and high bandwidth usage. Commercial platforms like Microsoft Teams Education Insights and Google Classroom Analytics rely on cloud-based processing with no privacy guarantees, and lack integration of behavioral analysis, adaptive networking, and secure decentralized frameworks in a unified architecture.

Research Gap

What's Missing

No existing system combines adaptive IoT environmental monitoring with privacy-preserving edge AI behavioral analysis, AI-driven SDN optimization, and blockchain-based zero-trust security in a single cohesive platform. Our system provides complete edge-based multi-modal analysis with mathematical privacy guarantees — a first in educational IoT research.

Research Problem

The Core Challenge

Designing a system that can monitor student well-being in real time while preserving privacy, optimizing network performance, and ensuring secure communication remains a key challenge in smart classroom environments — particularly in resource-constrained, developing-region educational institutions that rely on 2G/3G infrastructure.

Research Objectives

Goals

  • Design a multi-protocol IoT network for environmental and NFC-based attendance monitoring
  • Implement privacy-preserving edge AI for real-time behavioral and emotion analysis
  • Optimize classroom network performance using AI-driven SDN
  • Ensure secure access control using blockchain and AI-based anomaly detection

Methodology

How We Built It

IoT sensors (ESP32, MFRC522 RFID, DHT22, MQ-7, sound sensor) collect multi-modal environmental and attendance data at the physical layer. Edge processing runs directly on student endpoint devices — lightweight deep learning models (TensorFlow 2.17, DeepFace, OpenCV) perform local emotion and engagement inference, retaining only anonymized engagement statistics using Laplace differential privacy (ε=1.3) with 100% frame deletion before any data leaves the device. An SDN controller (Ryu + OpenFlow 1.3) manages campus network traffic using Logistic Regression flow classification to enforce DSCP-based QoS. A localized custom blockchain ledger (Python/Flask, SHA-256) performs Zero Trust device identity verification at 6.2ms median query latency. Anonymised engagement summaries are transmitted via HTTP POST to a Flask/SQLite educator dashboard.

A five-layer architecture enabling seamless data flow from physical sensors to the educator dashboard while maintaining privacy and security at every tier.

Key Technical Achievements

  • 100% frame deletion rate — 2,608 frames processed with zero biometric reconstruction risk
  • 0.20 Kbps per student vs. 72,000 Kbps for equivalent video streaming
  • 100 concurrent students consuming only 20 Kbps aggregate bandwidth
  • Blockchain device identity verification at 6.2ms median query latency, 847 queries/sec throughput
  • Multi-modal fusion: emotion (40%) + head movement (30%) + visibility (30%)

Designs and implements a three-layer edge AI engagement monitoring system using DeepFace and OpenCV exclusively on student endpoint devices. Applies Laplace differential privacy (ε=1.3) to 30-second aggregation windows with 100% frame deletion — achieving 84% engagement accuracy and a 358,293× bandwidth reduction vs. video streaming, with full GDPR-compliant biometric data protection.
